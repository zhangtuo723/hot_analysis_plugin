# 小红书爆款分析 Skill

## 适用场景
- 用户要求分析某个关键词在小红书上的爆款内容规律
- 需要获取热门笔记列表、互动数据，以及高赞笔记的深度素材，最终输出专业分析报告

## 总体流程

```
搜索关键词 → 抓取列表 → 筛选Top笔记 → 深度抓取 → 分析报告
```

---

## 阶段一：搜索与列表抓取

### 1. 定位搜索框
先用 `get_dom_structure` 查看页面 DOM，找到搜索输入框的 CSS 选择器。
- 常见选择器：`input[placeholder*='搜索']`、`#search-input`、`.search-input`
- 如果找不到，扩大 selector 到 `body` 再搜索

### 2. 输入关键词并搜索
使用 `type_text(selector, keyword, submit=true)` 输入关键词并提交。
- 工具内部会自动等待 4 秒让页面加载结果

### 3. 等待并确认结果页加载
调用 `screenshot` 查看搜索结果页是否已正常加载。
- 如果页面还在 loading，等待后再截图确认

### 4. 滚动加载更多笔记
使用 `scroll_page(pixels=800)` 向下滚动，每次滚动后工具会自动等待 2 秒。
- 目标：至少加载 30-50 篇笔记（通常需要滚动 3-5 次）

### 5. 提取列表数据
使用 `get_page_content` 提取笔记列表区域的文本内容。
- 先获取整个页面的文本，从中解析笔记标题、点赞数、收藏数、评论数、作者名
- 如果内容太杂乱，先用 `get_dom_structure` 找到笔记列表容器的选择器，再限定范围提取
- **尽量同时提取笔记链接或ID**（用于后续深度抓取时识别高赞笔记）

---

## 阶段二：Top 笔记深度抓取

### 6. 筛选高赞笔记
从阶段一的数据中，按**互动总量（点赞+收藏+评论）**排序，选出 **Top 3-5** 篇笔记。

### 7. 逐篇深度抓取（循环执行）
对每篇 Top 笔记依次执行：

**a. 进入详情页**
- 从列表页 `click` 点击该笔记的**封面图片**进入详情页（点击标题通常无效）
- 等待 2-3 秒后 `screenshot` 确认详情页已打开

**b. 提取正文内容**
- `get_page_content` 提取笔记正文全文
- 记录内容结构：开头钩子、正文节奏、结尾引导

**c. 截图记录视觉风格（处理多图轮播）**
- 第一张图：`screenshot` 记录当前显示的图片
- 检查是否为多图笔记：用 `get_dom_structure` 查看顶部图片区域是否有左右箭头或圆点指示器
  - **注意**：左右箭头可能是 hover 才显示的。如果看不到箭头，先用 `hover` 悬停到图片区域（常见选择器：图片容器 `[class*='slider']`、`.image-wrapper`、`.note-image`）触发显示
  - 常见箭头选择器：`[class*='arrow']`、`[class*='next']`、`button[class*='right']`、底部圆点 `[class*='dot']`
- 如果是多图轮播，循环执行：
  1. `hover` 悬停到图片区域确保箭头可见 → `click` 点击**右箭头/下一张按钮**
  2. 等待 0.5 秒后 `screenshot` 记录该图
  3. 重复直到无法继续切换（右箭头被禁用：`opacity` 降低 / `.disabled` / `aria-disabled=true`）或回到第一张
- 记录每张图的排版特点：纯图、拼图、文字覆盖、对比图等

**d. 视频笔记分析（如果是视频）**
- 如果详情页包含视频，使用 `extract_video_frames` 均匀抽帧分析画面风格、字幕设计、节奏
- 或使用 `capture_video_snapshot` 截取关键时间点画面

**e. 抓取评论区**
- `scroll_page(pixels=600)` 向下滚动到评论区
- `get_page_content` 提取热评内容（优先看点赞数高的评论）
- 记录用户的真实反馈、疑问、共鸣点

**f. 回到列表页**
- 优先尝试点击关闭按钮（常见选择器：`.close-btn`、`[class*='close']`、`.back`、`.icon-close`）
- 如果找不到关闭按钮，尝试点击页面左侧/右侧的遮罩区域（`.mask`、`.overlay`）
- 如果以上都失败，用 `type_text` 重新搜索关键词回到列表页
- 回到列表后 `screenshot` 确认位置

### 8. 进入下一篇
重复步骤 7，直到 Top 3-5 篇笔记全部抓取完成。

---

## 阶段三：爆款规律分析

基于阶段一的列表数据 + 阶段二的深度素材，从以下维度输出分析报告。

### 分析维度

#### 1. 宏观数据洞察（基于列表）
- **互动分布**：高赞笔记集中在哪些内容类型？
- **作者特征**：是头部博主还是素人更容易爆？
- **发布时效性**：近期热门 vs 长尾经典内容的比例

#### 2. 标题套路（基于列表）
- 常用句式（数字型、提问型、悬念型、身份标签型、对比型）
- 高频关键词和 emoji
- 标题长度分布
- **高互动标题 vs 低互动标题的差异**

#### 3. 封面风格（基于深度素材截图/视频抽帧）
- 配色倾向（暖色/冷色/高饱和/低饱和/莫兰迪）
- 文字排版（大字报、拼图、对比图、纯图）
- 人物出镜比例和表情特征
- **高赞笔记封面的共同点**

#### 4. 正文结构（基于深度素材）
- **开头钩子**：前 3 行怎么吸引人停留？（痛点直击、反常识、身份共鸣）
- **信息密度**：干货量是清单式、故事式还是教程式？
- **情绪曲线**：正文如何调动读者情绪？
- **结尾引导**：怎么引导点赞收藏评论？（CTA 话术）
- **段落节奏**：短句为主还是长文？ emoji 使用频率？

#### 5. 用户真实需求（基于评论区）
- **痛点挖掘**：用户在评论区问得最多的问题是什么？
- **决策疑虑**：用户在做购买/行动决策时担心什么？
- **情感共鸣**：哪些评论表达了强烈的共鸣、惊喜或焦虑？
- **内容缺口**：用户在评论里求但作者没讲到的内容

#### 6. 蓝海机会
- 热门关键词下尚未被充分覆盖的子话题
- 高互动但笔记数量少的细分方向
- 评论区用户强烈需求但现有笔记满足度低的方向

#### 7. 10个可复制爆款选题
- 每个选题包含：**标题建议** + **内容形式** + **封面方向** + **目标情绪**
- **必须基于真实数据推导**：
  - 标题句式模仿 Top 笔记中的高互动标题
  - 内容切入点来自评论区用户痛点
  - 封面风格参考高赞笔记的视觉特征
- 不要凭空编造，要说明「这个选题的依据是什么」

## 输出格式

用中文输出，结构清晰，重点突出。使用以下模板：

```markdown

- 扫描笔记数：N
- 深度分析笔记数：M
- 平均互动量：xxx
- 最高互动笔记：xxx（点赞/收藏/评论）

## 爆款共性
...

## 标题套路
...

## 封面风格
...

## 正文结构分析
...

## 用户真实需求（来自评论区）
...

## 蓝海机会
...

## 10个可复制选题
1. **标题**：...
   **依据**：基于 Top X 笔记的 xx 特征 + 评论区用户对 xx 的需求
   **形式**：...
   **封面**：...

2. ...
```

## 注意事项
- CSS 选择器必须使用标准语法（#id、.class、[attr='value']、:nth-child(n)），**不要使用 jQuery 的 `:contains('文本')`**
- 如果某个操作报错（如元素未找到），不要中断任务，换选择器或方法继续尝试
- 小红书搜索框通常是 contenteditable div，type_text 工具已兼容此场景
- 详情页可能是模态框弹窗，也可能是新页面，回到列表的方式需要灵活尝试
- 深度抓取时不要贪多，Top 3-5 篇足够支撑高质量的爆款分析
- 分析必须基于真实抓取的数据，不要编造数据
- 优先引用深度素材中的具体案例（某篇笔记的正文怎么写的、评论区哪条评论反映了什么需求）
- 如果数据量不足（列表少于10篇或深度素材少于3篇），提醒用户数据可能不够全面，建议补充抓取
- 分析要具体，不要泛泛而谈。与其说"标题要吸引人"，不如说"80%高赞标题使用了数字+身份标签的组合句式"
